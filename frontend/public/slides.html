The Problem

KYC verification today
creates risk for everyone.

235 crore Aadhaar e-KYCs in FY24. Every one collects, transmits, and stores full PII — creating liability across all three parties.

Users

Loss of Privacy

Full name, DOB, Aadhaar number, address, and photo shared with every verifier. No control over storage or downstream usage.

5+ platforms holding your complete identity
Companies

Compliance Burden

DPDP Act 2023 mandates consent, purpose limitation, storage limitation, breach notification. Every PII field stored is a liability.

Up to Rs 250 crore per breach under DPDP
Credential Issuers

Reputational Risk

When verifiers breach Aadhaar data, trust in the credential system erodes. UIDAI, DigiLocker — their credibility is at stake.

Breach at verifier = defamation of issuer
The Solution

Verify the claim.
Not the data.

Xylem is a zero-knowledge proof layer that sits between credential issuers and verifiers. The user generates a mathematical proof on their device. The verifier checks one equation and receives a boolean. No personal data is ever transmitted or stored.

Client-Side Proof Generation

Groth16 ZK-SNARK proofs generated in the browser using Circom circuits and snarkjs. Private inputs never leave the device.

~700ms proof generation

Configurable Verification

Unlike fixed claims (AgeAbove18), the verifier sets any threshold as a public input. One credential, any business rule.

age ≥ 21 • age 25-55 • state = TN

Zero PII Storage

The verifier stores only a Poseidon nullifier hash and a boolean result. If breached, the attacker gets meaningless numbers.

192 bytes stored • 0 bytes PII

Stack Selection Rationale

Groth16 — smallest proof size (192B), fastest verification (5ms). Ideal for API-based verification at scale.
BN254 — Ethereum precompile support; enables future on-chain anchoring without re-proving.
Circom + snarkjs — most mature ZK toolchain with browser-native WASM support. Client-side proving without native dependencies.
Poseidon — arithmetic-circuit-native hash. 100x fewer constraints than SHA-256 inside R1CS.

Implementation Highlights

SD-JWT Import — real Aadhaar VCs from Pehchaan parsed via Python extractor; claims surfaced for user approval before commitment.
QR Verification — presentation stored server-side with 3-min TTL; QR carries only an 8-char short ID for reliable phone scanning.
Multi-attribute — composite proofs combine N circuits with unified nullifier set via /verify/composite endpoint.
Async DB — background flush every 2s. Verification latency: ~50ms (crypto only, no disk blocking).

Implementation

How to Test

Live at eigenparse.com. Two demo accounts, two browser tabs, one cross-device QR flow.

Prover Flow (Tab 1)

1. Login as user@eigenparse.com / eigenparse
2. Select credential issuer (Manual or UIDAI SD-JWT)
3. Upload Aadhaar JWT or enter test data
4. Generate Poseidon commitment
5. Select attributes (age, aadhaar, state — multi-select)
6. Generate Groth16 proof(s) — ~700ms
7. QR code displayed with 3-min TTL
8. History tab — PDF receipt download

Verifier Flow (Tab 2 / Phone)

1. Login as verifier@eigenparse.com / zkproof
2. Click Scan QR — opens device camera
3. Point at prover's QR code
4. Payload fetched from server via short ID
5. snarkjs.groth16.verify() — ~5ms
6. Per-attribute pass/fail breakdown displayed
7. History tab — full audit trail with PDF export
8. Works as installed PWA (Add to Home Screen)

Use Cases

Sector Applications

Lending & NBFC

Loan Pre-qualification

age 25-55 AND income range AND credit threshold. One composite proof, zero data. Nullifiers detect duplicate applications across platforms.

Telecom

SIM KYC

DoT-compliant activation using Aadhaar validity proof. No photocopies, no biometric capture, no agent dependency. Instant provisioning.

Government

Subsidies & Scholarships

State residence + age bracket in one proof. Nullifier dedup prevents cross-district duplicate claims. Replaces 5+ document submissions.

Insurance

Bracket Pricing

Range proof reveals age bracket (25-35, 35-45) without DOB. Composite with metro residence for urban health plan eligibility.

Property & Rental

Tenant Verification

Aadhaar (identity) + ITR (income ≥ 3x rent) + EPFO (employed). Landlord receives one boolean. Zero document copies.

KYC Dedup

Cross-Platform Identity

Same person at 5 banks = same nullifier. Banks share nullifier registry (not PII). Fraud detection without centralized identity database.
